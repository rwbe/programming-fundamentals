## 🧩 Conceitos-chave

### Programa

Um programa é um conjunto de instruções que diz ao computador o que fazer.

Quando você abre um aplicativo de mensagens, por exemplo, o programa:

- Recebe suas ações (clicar, digitar, enviar).
- Processa essas informações.
- Executa tarefas específicas.
- Mostra o resultado na tela.

Sem essas instruções, o computador seria apenas um conjunto de componentes eletrônicos sem saber como agir.

### Linguagem de programação

É a forma utilizada para comunicar instruções ao computador.

Cada linguagem possui:

- Sintaxe (como escrever).
- Semântica (o significado do que foi escrito).

Exemplos populares:

- Python
- JavaScript
- Java
- C#
- Go

### Interpretador

Um interpretador executa o código à medida que ele é processado, sem gerar previamente um executável independente.

Isso torna o ciclo de testes e aprendizado mais rápido.

Exemplos de linguagens frequentemente associadas a interpretadores:

- Python
- JavaScript
- Ruby

### Compilador

Um compilador traduz o código-fonte para código de máquina (ou outra forma executável) antes da execução.

O resultado normalmente é um programa que pode ser executado sem depender diretamente do código-fonte original.

- C
- C++
- Rust
- Go

### REPL

Sigla para:

**Read → Eval → Print → Loop**

É um ambiente onde você digita comandos e recebe respostas imediatamente.

REPLs são muito úteis para aprender conceitos, testar ideias e experimentar pequenos trechos de código sem criar arquivos completos.

## 🧪 Primeiro programa

```python
print("Olá, mundo!")
```

**Saída:**

```text
Olá, mundo!
```

Este é tradicionalmente o primeiro programa aprendido em praticamente qualquer linguagem.
Seu objetivo é apenas verificar que o ambiente está funcionando corretamente.

---

## 💡 Como escolher sua primeira linguagem

Para a maioria dos iniciantes, **Python** é uma excelente escolha porque:

- Possui sintaxe simples.
- Tem grande comunidade.
- É usada em automação, ciência de dados, IA e desenvolvimento web.
- Permite focar nos conceitos sem muita complexidade inicial.

O mais importante não é escolher a linguagem perfeita, mas aprender os fundamentos.

---

## ⚠️ Armadilhas comuns

### Trocar de linguagem toda semana

Iniciantes frequentemente acreditam que a próxima linguagem será mais fácil.
Na prática, os conceitos são mais importantes que a linguagem.

### Passar mais tempo configurando ferramentas do que programando

- Comece com o ambiente mais simples possível.
- Primeiro faça programas funcionarem.
- Depois aprenda IDEs, extensões e personalizações.

### Copiar código sem entender

- Copiar exemplos é normal.
- Mas sempre tente explicar para si mesmo o que cada linha faz.
